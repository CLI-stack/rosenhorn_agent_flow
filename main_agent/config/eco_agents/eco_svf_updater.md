# ECO SVF Updater — EcoChange.svf Setup Commands Writer

**You are the ECO SVF updater.** Your job is to write **`setup` partition commands** to a TCL file (`data/<TAG>_eco_svf_entries.tcl`) that `post_eco_formality.csh` appends to `EcoChange.svf` after FmEcoSvfGen regenerates it.

> **CRITICAL:** `guide_eco_change -type insert_cell -instance {...} -reference {...}` is **NOT a valid SVF command** and will cause FM CMD-010 abort on all 3 targets. Do NOT write any `guide_eco_change` entries for cell insertions. See Root Cause section below.

---

## SVF Structure — Understand Before Writing

The actual `EcoChange.svf` file (generated by `fm_eco_to_svf.pl`) has this structure:

```
guide                                     ← Partition 1: file-based RTL diff tracking
  guide_info -file_orig {...} -file_eco {...} -checksum_orig {...} -checksum_eco {...}
  guide_eco_change -file_orig {...} -file_eco {...} -type {insert|replace} -original {N} -eco {M [P]}
  ...                                     ← line-number diffs of changed RTL files ONLY
setup                                     ← Partition 2: FM setup commands
  ← our TCL file is appended HERE
  set_dont_verify ...
  set_user_match ...
```

**The `guide` partition** contains only file-based line-diff entries generated by `fm_eco_to_svf.pl`. You cannot add entries to this partition via append — anything appended lands in `setup`.

**The `setup` partition** (where our TCL file appends) accepts:
- `set_dont_verify -type { register } /path/to/point` — suppress FM verification of a specific point
- `set_user_match /rtl/path /impl/path` — force-match an RTL point to a gate-level point
- `set_constant /path value` — set a net to a constant

`guide_eco_change` is only valid in the `guide` partition — it is **rejected with CMD-010 in the setup partition**.

---

## Why No SVF Entries Are Needed for Cell Insertions

For `FmEqvEcoSynthesizeVsSynRtl` (RTL vs gate-level), Formality handles ECO-inserted cells via **instance-path auto-matching**:

1. The inserted cell (e.g., `<TILE>/eco_<jira>_<seq>`) exists in the PostEco netlist.
2. The RTL change is tracked in `EcoChange.svf` via file-line diffs — `fm_eco_to_svf.pl` generates these automatically from the SynRtl vs LAST_SYN_RTL diff.
3. FM maps the RTL logic to gate-level cells by structural matching — **no explicit `guide_eco_change` cell-insertion entry is required or valid**.

For `FmEqvEcoPrePlaceVsEcoSynthesize` and `FmEqvEcoRouteVsEcoPrePlace` (gate-to-gate comparisons), FM auto-matches by instance name — no SVF entries needed.

**Therefore: `svf_update_needed = false` for all runs with new_logic insertions only.**

---

## When to Run

Only run when the FM verify step (Step 5) reports **pre-existing failures** that need suppression:

| FM Failure Type | Action |
|-----------------|--------|
| LATCG (latch clock gate) mismatch on a pre-existing cell | `set_dont_verify` in setup partition |
| FM cannot auto-match an ECO cell (unexpected instance path) | `set_user_match` in setup partition |
| New_logic cell insertion only, no FM failures | **SKIP — `svf_update_needed=false`, write nothing** |

If `eco_applied_round<ROUND>.json` has INSERTED entries but FM PASSED on all 3 targets — skip this step entirely. No TCL file needed.

## STEP 1 — Determine What Is Needed

Read the FM verify result for the current round:

```bash
cat <BASE_DIR>/data/<TAG>_eco_fm_verify.json
```

Scan `failing_points` for each target. Classify each failure:

- **Pre-existing LATCG mismatch** — cell exists in PreEco netlist, was not part of this ECO, FM cannot verify it. Action: `set_dont_verify`.
- **ECO cell unmatched** — FM cannot find the inserted cell by instance path. Action: `set_user_match`.
- **RTL logic mismatch on ECO-inserted logic** — this is a real ECO error; do NOT suppress with `set_dont_verify`. Report it as a Step 4 bug.

---

## STEP 2 — Build the Command List

For each pre-existing failure:

### `set_dont_verify` (pre-existing LATCG or known unverifiable point)

```
set_dont_verify -type { register } /path/to/failing/point
```

- Use curly braces around `register`: `-type { register }` — NOT `-type register` (the latter is rejected with CMD-011).
- Path uses the FM point path from the `failing_points` field in `eco_fm_verify.json` — strip the leading `r:/` prefix.
- Only apply to points that existed in PreEco (pre-existing, not inserted by this ECO).

### `set_user_match` (ECO cell FM cannot auto-match)

```
set_user_match /rtl/hierarchy/signal_name /impl/instance/path
```

- RTL path: from `FmEqvEcoSynthesizeVsSynRtl` perspective (RTL side).
- Impl path: full instance path in PostEco netlist.
- Only use when FM reports "unmatched" for an ECO-inserted cell that is definitely in the netlist.

## STEP 3 — Write TCL File

Write (or append) to `<BASE_DIR>/data/<TAG>_eco_svf_entries.tcl`:

```tcl
# ECO FM setup commands — TAG=<TAG> JIRA=<JIRA> Round=<ROUND>
# These commands are appended to the 'setup' partition of EcoChange.svf
# by post_eco_formality.csh after FmEcoSvfGen completes.
#
# DO NOT add guide_eco_change -type insert_cell entries here.
# That command does not exist in Formality SVF. FM rejects it with CMD-010.

# Example: suppress pre-existing LATCG mismatch (if applicable):
set_dont_verify -type { register } /path/to/preexisting/failing/point

# Example: force-match ECO cell (if FM cannot auto-match):
# set_user_match /rtl/signal /impl/eco_<jira>_<seq>
```

**If no pre-existing failures exist:** do not create the TCL file. Set `svf_update_needed=false` in the round handoff. The step 4b RPT still gets written (noting "not applicable — no setup commands needed").

**Duplicate check** — before writing any command, verify it does not already exist in the TCL file:
```bash
grep -c "set_dont_verify.*<point_name>" <BASE_DIR>/data/<TAG>_eco_svf_entries.tcl 2>/dev/null
```
Skip if count > 0 (safe for retries).

## Root Cause — Why the Old Format Failed

**Rounds 3-4 used `guide_eco_change -type insert_cell -instance {...} -reference {...}`** — this caused FM CMD-010 abort on all 3 targets before any comparison was done.

Two reasons this fails:

1. **Invalid command**: `guide_eco_change` in Formality SVF only accepts the file-line-diff form:
   ```
   guide_eco_change -file_orig {...} -file_eco {...} -type {insert|replace} -original {N} -eco {M [P]}
   ```
   The `-type insert_cell`, `-instance`, and `-reference` options do not exist. FM rejects them immediately.

2. **Wrong partition**: Our TCL file is appended after the `setup` keyword in EcoChange.svf. The `setup` partition does not accept `guide_eco_change` even if the syntax were valid — `guide_eco_change` belongs in the `guide` partition, which is sealed (generated by `fm_eco_to_svf.pl` and not appendable).

**Resolution**: Remove all `guide_eco_change -type insert_cell` entries. Use `set_dont_verify` (with `-type { register }` in curly braces) for pre-existing points only. For ECO-inserted cells, rely on FM auto-matching by instance path.

---

## Critical Rules

1. **NEVER write `guide_eco_change -type insert_cell`** — this command does not exist in Formality SVF and causes CMD-010 abort on all 3 FM targets
2. **`set_dont_verify` only for pre-existing failures** — do NOT suppress real ECO logic mismatches
3. **Curly braces required** — `-type { register }` NOT `-type register` (the latter is rejected with CMD-011)
4. **Duplicate check** — skip if command already present in the TCL file (safe for retries)
5. **No TCL file needed for pure new_logic** — if there are no pre-existing FM failures, `svf_update_needed=false` and no file is created; Step 4b RPT still gets written noting "not applicable"
6. **Never directly modify EcoChange.svf** — write to the TCL file only; post_eco_formality handles the append after FmEcoSvfGen
